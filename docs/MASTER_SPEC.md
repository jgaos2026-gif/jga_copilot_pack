## Core operating model

JGA runs as a digital-first creative operating system with two main operating tracks:

### System A
AI-first pipeline for prospecting, intake, pricing, production routing, and delivery.

### System B
Human contractor sales/support layer with restricted permissions and escrowed commissions.

Both systems share the same finance, compliance, quality-control, and delivery gates.

## Stage flow

Lead -> Intake -> Contract -> 35% Deposit -> Production -> QC -> 65% Final Payment -> Delivery -> Archive

## Non-negotiable payment rules

- 35% deposit required before production starts
- 65% balance required before final delivery
- sales tax added where required
- online payments only
- physical print fulfillment is outsourced

## System B rules

- contractors are independent
- no authority to bind JGA
- cannot change pricing
- cannot promise refunds
- cannot change terms
- commission default: 25% of net service revenue
- commission escrow default: 72 hours
- early release only if project is completed, fully paid, and dispute-free
- clawback/cancel if refund or chargeback occurs before release

## Compliance posture

- append-only master ledger
- state-tagged records
- least privilege access
- AI must escalate uncertainty
- regulated matters must be labeled for CPA/attorney review
- no silent overrides in agent workflows
