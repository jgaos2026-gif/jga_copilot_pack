## Reporting a Vulnerability

**Do NOT open a public GitHub issue for security vulnerabilities.**

If you discover a security vulnerability in JGA Enterprise OS, please
report it responsibly by emailing:

**security@jaysgraphicarts.com**

Include in your report:
- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact assessment
- Any suggested remediation (optional)

We will acknowledge your report within **48 hours** and aim to provide a
full response and remediation timeline within **7 business days**.

## Disclosure Policy

- We follow a coordinated disclosure model.
- Security issues will be patched before any public disclosure.
- Reporters who follow responsible disclosure will be acknowledged
  (unless anonymity is requested).

## Scope

This policy covers all code and infrastructure in this repository,
including but not limited to:

- Authentication and authorization flows
- Pricing and financial calculation logic
- Contract and ledger data handling
- API endpoints and server actions
- Supabase Row-Level Security policies

## Out of Scope

- Denial-of-service attacks
- Issues in third-party dependencies (report those upstream)
- Social engineering attacks
